# agentsmith-skills

Versioned skill and pattern definitions for [agent-smith](https://github.com/holgerleichsenring/agent-smith).

## Layout

```
agentsmith-skills/
├── skills/                       # Role definitions (api-security, coding, legal, mad, security)
│   ├── concept-vocabulary.yaml   # Shared vocabulary referenced by activation keys (v2)
│   ├── _index/                   # Build artifact (generated by SkillIndexBuilder, gitignored)
│   └── <category>/
│       └── <role>/
│           ├── SKILL.md          # Required — extended frontmatter (v2) + role bodies
│           ├── agentsmith.md     # Legacy orchestration metadata (read by old loader)
│           ├── source.md         # Provenance: origin, version, reviewed
│           └── references/       # Optional reference files cited via {{ref:<id>}}
└── patterns/                     # YAML regex pattern definitions for the static scanner
    ├── ai-security.yaml
    ├── api-auth.yaml
    ├── auth.yaml
    ├── compliance.yaml
    ├── config.yaml
    ├── injection.yaml
    ├── secrets.yaml
    └── ssrf.yaml
```

## Extended Frontmatter (v2)

Skills declare **roles_supported**, **activation criteria**, **role assignment
criteria**, **references**, and an **output contract** in their SKILL.md
frontmatter. Triage assigns each skill a role and pipeline phase per ticket.

The four roles:

| Role     | Phase  | Output         | When applied                                                                  |
|----------|--------|----------------|-------------------------------------------------------------------------------|
| lead     | Plan   | plan           | Sets the architectural / data / security plan reviewers compare against later |
| analyst  | Plan   | list           | Contributes perspective to the lead's plan, no veto                           |
| reviewer | Review | list           | Compares actual code/diff against the plan, evidence-required                 |
| filter   | Final  | list \| artifact | Reduces a finding list to true positives, or synthesizes a final-report artifact |

A single skill (e.g. architect) may declare multiple supported roles and be
assigned different roles in different tickets. Activation positives reference
a controlled vocabulary in `skills/concept-vocabulary.yaml`; activation
negatives and role-assignment criteria are skill-local.

Canonical example: [skills/coding/architect/SKILL.md](skills/coding/architect/SKILL.md).
Three pattern anchors:
- **architect** — supports all three discussion roles (lead/analyst/reviewer)
- **tester** — analyst+reviewer with phase differentiation, no lead
- **false-positive-filter** — filter-only with `output: list`

Triage, the new SkillLoader, and the strict frontmatter validator land in
agent-smith via phases p0111a / p0111c. Until then the legacy SkillLoader
keeps reading these skills via its existing tolerant YAML parser — fields it
does not know are simply ignored, so the running pipeline is unaffected.

Claude Sonnet 4.6 is the primary tuning target. Other providers
(Claude Haiku/Opus, OpenAI, Gemini, Ollama) are supported best-effort. A skill
may ship a `SKILL.<provider>.md` override (e.g. `SKILL.openai.md`) to tune
for a specific provider; if absent, the base `SKILL.md` is used.

For the full migration guide see
[docs/configuration/skills/migration.md](https://docs.agent-smith.org/configuration/skills/migration/)
(written when p0111c lands; until then the canonical reference is the
architect skill).

## How agent-smith consumes this

The agent-smith server resolves content at boot via `SkillsBootstrapHostedService`,
driven by the `skills:` section in `agentsmith.yaml`:

```yaml
skills:
  source: default      # default | path | url
  version: v1.0.0      # default-source: which release to pull
  cacheDir: /var/lib/agentsmith/skills  # where to extract
  # path: /opt/skills          # path source: pre-mounted directory
  # url: https://...           # url source: explicit tarball URL
  # sha256: <hex>              # url source: SHA256 verification
```

The `agentsmith skills pull` CLI command runs the same download/verify/extract
path standalone — see [docs/skills/source-modes.md](https://github.com/holgerleichsenring/agent-smith/blob/main/docs/skills/source-modes.md).

## Releases

Each tag of the form `vX.Y.Z` produces:

- `agentsmith-skills-vX.Y.Z.tar.gz` — deterministic tarball of `skills/` (and from
  v2.0.0 onwards, also `patterns/`)
- `agentsmith-skills-vX.Y.Z.tar.gz.sha256` — SHA256 sidecar

The tarball is reproducible: any rebuild of the same commit produces a byte-equal
archive. See [scripts/package.sh](scripts/package.sh).

The `edge` tag tracks `main` and is republished on every push.

## Air-gap installs

Pre-download a release in a connected environment, then point the server at it:

```bash
agentsmith skills pull --version v1.0.0 --output /var/lib/agentsmith/skills
# or in airgap, copy the tarball + sha256 across the boundary, then:
agentsmith skills pull --url file:///path/to/agentsmith-skills-v1.0.0.tar.gz \
                      --sha256 $(cat agentsmith-skills-v1.0.0.tar.gz.sha256)
```

Or override the default repository URL without changing config:

```bash
export AGENTSMITH_SKILLS_REPOSITORY_URL=https://internal.mirror/agentsmith-skills
```

## Contributing

- New skill: add `<category>/<role>/SKILL.md` with the required frontmatter.
- New pattern: add an entry to one of `patterns/*.yaml` or create a new file there.
  Pattern IDs are opaque — see the
  [custom-patterns docs](https://github.com/holgerleichsenring/agent-smith/blob/main/docs/security/custom-patterns.md).
- Bumping the catalog: edit `CHANGELOG.md`, tag `vX.Y.Z`, push the tag.
